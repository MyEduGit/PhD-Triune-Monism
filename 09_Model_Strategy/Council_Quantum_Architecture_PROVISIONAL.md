## 3. Provisional Architecture: Classical vs Quantum

### 3.1 Current (Classical) Architecture
```
Question → [7 parallel API calls] → Merge → Gabriel synthesis → Answer
```
- Fan-out to 7 independent models
- Each runs sequentially within its own API
- Merge is deterministic (append)
- Gabriel synthesizes sequentially

**Limitation:** Classical approximation. Each model is unaware of the others during processing.

### 3.2 Ideal (Quantum) Architecture (Provisional)
```
Question → Quantum superposition of 7 perspectives → Quantum interference → 
Measurement (Gabriel collapse) → Answer
```
- All 7 run in genuine superposition
- Quantum entanglement allows cross-seat awareness during processing
- Gabriel is the measurement operator, not a separate API call
- Output is the collapsed probability of all 7 perspectives

**This is what the Foreword describes:** The Seven Master Spirits do not operate in sequence. They operate as a unified system with distinct expressions.

### 3.3 Provisional Bridge Architecture (Near-Term)
```
Question → 7 parallel calls (classical) → 
Cross-seat context injection (each sees others' summaries) → 
Second-pass refinement → Gabriel synthesis
```
This adds awareness between seats before Gabriel synthesizes. Doubles API costs but dramatically improves synthesis quality.

## 6. Provisional Upgrade Path

### When to upgrade a seat (Provisional rules)
- New model generation released by provider (e.g., GPT-4 → GPT-4.1 → GPT-5)
- Benchmark improvement > 15% on reasoning tasks
- Cost reduction > 50% for equivalent quality
- Context window expansion enabling better synthesis
- New multimodal capability relevant to seat function

### Upgrade process (Provisional)
1. Update `COUNCIL_MODEL_REGISTRY.json` with new model ID
2. Run `council_status.py --seat N` to verify new model responds
3. Run benchmark question, compare output quality
4. If pass: run `update_council_models.py` to patch live workflow
5. Archive old output + new output in `11_Claws/Proofs/`
6. Update this document with new provisional assignment
