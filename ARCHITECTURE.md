## Purpose

One shared Obsidian vault receives the past, present, and future inputs /
outputs of four AI systems, with all content types (text, images,
screenshots, files, audio, binaries). The vault is the **primary source of
truth**. GitHub, Notion, and Zapier are **mirrors / fan-outs**.

## Four AI Sources

| Source      | Transport                                | Notes                              |
|-------------|------------------------------------------|------------------------------------|
| ChatGPT     | Official export (conversations.json) + Web Clipper / API where available | "PR #1" alias in planning docs |
| Claude Code | This session + `claude.ai/code` exports + MCP GitHub writes | Current actor                     |
| Grok        | xAI API + manual export                  | No official bulk export yet        |
| Gemma4      | Local runtime (Ollama / llama.cpp) log hook | Writes directly to inbox/ on host  |

## Canonical Vault Layout

All four AIs write to the same folder shape inside
`~/Documents/Obsidian/PhD-Triune-Monism/` (mirrored to this repo):

```
<YYYY-MM-DD>/<job-slug>/
  job.md                          # narrative + links (primary note)
  assets/
    <job>.mmd                     # Mermaid source
    <job>.svg                     # rendered (generated by validate.sh)
    <job>.png                     # rendered (generated by validate.sh)
    <job>-canva.png               # optional, Canva MCP output
    raw/                          # pics, screenshots, audio, binaries
    transcripts/                  # per-AI chat dumps (md or json)
  scripts/
    install.sh                    # wire DOC skill + hooks for this job
    validate.sh                   # render diagrams, check links, fail loud
  hooks/
    doc-reminder.sh               # nudges when job.md is stale
    doc-verify.sh                 # enforced pre-commit: all links resolve
```

Each AI dumps its chat turns under `assets/transcripts/<ai>.md` (or `.json`
for structured exports). Binary artifacts go in `assets/raw/` with a README
index in `job.md`.

## DOC Skill

`/doc <job-slug>` is the single entry point. It:

1. Creates `<today>/<job-slug>/` with the layout above.
2. Seeds `job.md` from a template (YAML frontmatter: `ai_sources`, `date`,
   `status`, `notion_url`, `github_commit`).
3. Writes `assets/<job>.mmd` from a diagram prompt.
4. Runs `scripts/validate.sh` → Mermaid → SVG + PNG (via `mmdc`).
5. Optionally calls Canva MCP to produce `<job>-canva.png`.
6. Commits the folder to the active feature branch.
7. Creates a Notion page under the Sovereign Dashboard embedding the public
   diagram URL and inlining `job.md`.
8. If `ZAPIER_WEBHOOK_URL` is set, posts a fan-out payload (daily digest,
   Drive backup, calendar reminder).
9. Appends a node + `click` link to `master_pipeline.mmd`.

Hooks:

- `hooks/doc-reminder.sh` — cron / Claude Code SessionStart hook that
  checks for jobs with `status: draft` > 24h old.
- `hooks/doc-verify.sh` — pre-commit gate: every `job.md` must have a
  rendered SVG sibling and a non-empty GitHub commit reference in frontmatter.

## Flow

```
AI source  →  inbox/  →  DOC skill  →  Obsidian vault
                                       ├─→ GitHub commit
                                       ├─→ Notion page (Sovereign Dashboard)
                                       └─→ Zapier fan-out (optional)
```

See `master_pipeline.mmd` for the authoritative interactive diagram.

## The Lucifer Test (per UrantiOS)

Every component must answer yes to all four:

- Transparent — all writes are git-logged and link back to source.
- Honest — failures are recorded in `job.md`, not hidden.
- In mandate — no AI writes outside its own `assets/transcripts/<ai>.md`.
- Serves the mission — if a job doesn't advance the Foreword, it gets a
  `status: archival` tag and is excluded from the master dashboard.

## Known Gaps (Truth Report)

- Canva, Notion, and Zapier wiring are declared but not executed in this
  commit. The dry-run job at `2026-04-14/multi-ai-obsidian-integration-dryrun/`
  records which steps ran and which are pending.
- Gemma4 log hook and Grok bulk-export path are specified, not yet
  implemented.
- Replication to `mircea-constellation`, `lobsterbot`, `URANTiOS` repos is
  deferred until the PhD-Triune-Monism dry-run validates end-to-end.
